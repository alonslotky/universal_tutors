Handling Database Changes

In this project, South should be used to handle all database schema migrations. Below are some of the most common tasks you might need to peform. For access to the full South documentation, please click here:

  • I've create a brand new app in the apps/ folder (i.e. apps/someapp)
    run 'sh manage schemamigration someapp --initial';
    run 'sh manage migrate someapp';

    There is a chance that it will error if you have defined custom fields that are unknown to South. To resolve this, simply follow the instructions here http://south.aeracode.org/wiki/MyFieldsDontWork

  • I've made changes to model(s) in an already existing app:
    run 'sh manage schemamigration someapp --auto'
    run 'sh manage migrate someapp'

  • Changes have been made to a model or models, and i need to update my database to reflect these changes:
    To update your database with the latest migrations run: sh manage migrate someapp
